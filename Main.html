Git Topic

Git adalah salah satu dari Version Control System (VCS). Version Control System adalah sistem yang menyimpan rekaman/snapshot perubahan pada source code, memungkinkan bekerja berkolaborasi dengan lebih baik, mengetahui siapa yang melakukan dan kapan sebuah perubahan terjadi, dan memungkinkan kita untuk kembali ke keadaan sebelum perubahan (checkout). Git adalah sebuah software untuk mengelola perusahaan file di dalam folder. File di dalam folder disebut Repository/Repo. Sedangkan Github adalah website/layanan cloud untuk menyimpan dan mengelola project/repoÂ git.

Basic Git Commands

  1. git init: untuk repositori git yang kosong atau inisialisasi ulang repositori yang sudah ada.
  2. git status: untuk memeriksa status git.
  3. git log: untuk melihat catatan log perubahan pada repositori.
  4. git add: untuk menambahkan konten file ke indeks.
  5. git commit: untuk mencatat perubahan pada repositori.
  6. git push: untuk mengirim perubahan dari repositori lokal ke repositori gift jarak jauh.

Git Branching

  1. git branch: untuk melihat daftar, membuat cabang, dan menghapus cabang.
  2. git fetch: untuk mengambil perubahan dari repositori di jarak jauh ke repositori lokal.
  3. git checkout: untuk berpindah antara cabang atau untuk memeriksa berkas di berbagai titik waktu dalam repositori.
  4. git clone: untuk membuat salinan dari repositori git yang ada dari server atau repository jarak jauh ke direktori lokal.
  5. git pull: untuk mengambil perubahan dari repositori git jarak jauh ke repositori lokal.
